Product Cipher

Advanced encryption by combining Substitution and Transposition ciphers

📘 How Product Cipher Works

A Product Cipher is a powerful hybrid encryption technique that applies a Substitution Cipher followed by a Transposition Cipher in multiple rounds for significantly stronger security.

Step 1: Substitution
Replaces each letter using chosen method (Caesar, Vigenère, Playfair, Hill, etc.)
Step 2: Transposition
Rearranges the order of letters using Rail Fence or Columnar technique
Step 3: Multiple Rounds
The output of one round becomes the input for the next round
Mathematical Flow:
C = Tn( Sn( Tn-1( Sn-1( ... S1(P) ... ))))
Where P = Plaintext, S = Substitution, T = Transposition, C = Ciphertext

Caesar Cipher

One of the simplest and most widely known encryption techniques

📘 How Caesar Cipher Works

The Caesar Cipher is one of the simplest and most widely known encryption techniques. It's a substitution cipher where each letter in plaintext is shifted a fixed number of positions down the alphabet.

🔐 Encryption Formula

Ci = (Pi + k) mod 26

Where C = Ciphertext, P = Plaintext, k = Shift value, i = Position in text

🔓 Decryption Formula

Pi = (Ci - k + 26) mod 26

📋 Algorithm Steps

Encryption Process:
  1. Input plaintext P and shift k
  2. Convert letters → numbers (A=0, B=1, ..., Z=25)
  3. For each letter:
    Ci = (Pi + k) % 26
  4. Convert numbers → letters
  5. Output ciphertext
Decryption Process:
  1. Input ciphertext C and shift k
  2. Convert letters → numbers (A=0, B=1, ..., Z=25)
  3. For each letter:
    Pi = (Ci - k + 26) % 26
  4. Convert numbers → letters
  5. Output plaintext

💡 Example

Plaintext: "HELLO"

Shift: 3

H (7) + 3 = 10 → K
E (4) + 3 = 7 → H
L (11) + 3 = 14 → O
L (11) + 3 = 14 → O
O (14) + 3 = 17 → R

Ciphertext: "KHOOR"

Vigenère Cipher

A method of encrypting alphabetic text by using a series of interwoven Caesar ciphers

📘 How Vigenère Cipher Works

The Vigenère Cipher uses a keyword to determine the shift value for each letter, making it much more secure than a simple Caesar cipher.

Encryption: C = (P + K) mod 26
Decryption: P = (C - K + 26) mod 26
Where P = Plaintext, C = Ciphertext, K = Key letter value

Monoalphabetic Cipher

A substitution cipher where each letter maps to exactly one other letter

📘 How Monoalphabetic Cipher Works

The Monoalphabetic Cipher uses a fixed substitution alphabet where each letter of the plaintext is mapped to a unique letter in the ciphertext alphabet.

Playfair Cipher

A digraph substitution cipher that encrypts pairs of letters

📘 How Playfair Cipher Works

The Playfair Cipher encrypts pairs of letters (digraphs) using a 5x5 key matrix, making it significantly more secure than simple substitution ciphers.

Hill Cipher

A polygraphic substitution cipher based on linear algebra

📘 How Hill Cipher Works

The Hill Cipher uses matrix multiplication to encrypt blocks of letters, making it one of the first polygraphic ciphers.

One-Time Pad

Theoretically unbreakable encryption when used correctly

📘 How One-Time Pad Works

The One-Time Pad is a symmetric encryption technique that uses a random key that is at least as long as the message and is used only once, making it mathematically unbreakable when used correctly.

🔐 Encryption Formula

Ci = (Pi + Ki) mod 26

Where C = Ciphertext, P = Plaintext, K = Key letter value, i = Position in text

🔓 Decryption Formula

Pi = (Ci - Ki + 26) mod 26

📋 Algorithm Steps

🔐 Encryption Process:
  1. Take plaintext P and key K
  2. Convert both to uppercase
  3. Remove spaces (optional for your app)
  4. Check length requirement:
    If length(P) ≠ length(K)
    Show error → "Key must be same length as plaintext"
  5. Convert letters to numbers:
    A = 0, B = 1, ..., Z = 25
  6. For each letter:
    Ci = (Pi + Ki) % 26
  7. Convert result back to letters
  8. Display ciphertext C
🔓 Decryption Process:
  1. Take ciphertext C and same key K
  2. Convert both to uppercase
  3. Convert letters to numbers:
    A = 0, B = 1, ..., Z = 25
  4. For each letter:
    Pi = (Ci - Ki) % 26
  5. If result is negative:
    Add 26
  6. Convert back to letters
  7. Display original plaintext P

💡 Example

Plaintext: "HELLO"

Key: "XMCKL"

H (7) + X (23) = 30 → 4 → E
E (4) + M (12) = 16 → Q
L (11) + C (2) = 13 → N
L (11) + K (10) = 21 → V
O (14) + L (11) = 25 → Z

Ciphertext: "EQNVZ"

🔒 Security Properties

  • Perfect Secrecy: Key is random and used only once
  • Information-Theoretic Security: Unbreakable without key knowledge
  • Key Management: Critical - key must be securely shared
  • Key Reuse: NEVER reuse keys - destroys security

Rail Fence Cipher

A transposition cipher that writes the message in a zigzag pattern

📘 How Rail Fence Cipher Works

The Rail Fence Cipher writes the plaintext in a zigzag pattern across multiple "rails" and then reads off the rows to create the ciphertext.

Columnar Transposition

A transposition cipher that rearranges the plaintext by columns

📘 How Columnar Transposition Works

The Columnar Transposition writes the plaintext in rows of a defined width and then reads off the columns in the order specified by the keyword.
